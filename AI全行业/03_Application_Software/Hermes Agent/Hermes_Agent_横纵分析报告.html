一、一句话定义

Hermes Agent 是 Nous Research 在 2026 年 2 月推出的开源 AI Agent 框架，核心定位是"会随使用而成长的智能体"：它能够自动从任务经验中生成可复用技能、维护四层长期记忆、跨平台持续工作，代表了 AI Agent 从"会话工具"向"持续工作者"演进的一个方向。

二、纵向分析：Hermes Agent 是怎么来的，它解决了什么问题？

2.1 它不是无中生有——先理解 Nous Research 是谁

要理解 Hermes Agent，得先知道做它的团队。

Nous Research 是一家 2023 年在纽约成立的 AI 公司，团队不大（10-30 人规模），但在开源大模型圈早就有积累。

它最知名的产品，恰好也叫 Hermes —— 一个系列的开源微调模型：Hermes 2、Hermes 3、Hermes 3 on Qwen 2.5 等等。这些模型的特点是对指令跟随、工具调用和结构化输出做了专门优化，在 Hugging Face 上累计下载量超过 5500 万次，被 120 多个应用集成。

这是一个很重要的背景。Nous Research 不是一家突然跑出来做 Agent 的新公司，而是先在模型层建立了开发者信任，再往上走一层。

融资方面，它完成了种子轮 520 万美元，随后拿到 A 轮 5000 万美元，由 Paradigm 领投，估值约 10 亿美元。Paradigm 是 Web3 / 加密领域最知名的风投之一，这也带来了一个后来的争议点（我们后面会讲）。

所以 Hermes Agent 一出来，不是冷启动状态。它自带了 Nous Research 模型生态里积累的开发者社区。

2.2 它出现在一个什么样的时间点？

2026 年初，AI Agent 赛道已经不是纸上谈兵了。

OpenClaw（GitHub 上超过 35 万星，中文圈俗称"龙虾"）已经把"本地部署的个人 AI 助手"这个概念做成了现象级产品，很多人通过它第一次体验到了"AI 不只是聊天框，还可以接微信、接日历、接代码仓库"的感觉。

但 OpenClaw 火起来之后，一部分开发者也开始感受到这类产品的共同局限：

  • 每次对话结束，AI 对"这次发生了什么"几乎没有记忆
  • 今天教它一个做事方法，明天又要重新教
  • 上下文窗口撑满之后，它开始忘事
  • 它很聪明，但不会"进步"

Nous Research 在这个背景下推出 Hermes Agent，主打的就是这些问题的解法。

它在官方说明里写得很直接：现有 Agent 的四个核心痛点是——
1. 记忆断层：对话结束记忆消失
2. 能力固化：无法从经验中学习新技能
3. 上下文爆炸：随对话增长性能下降
4. 部署复杂：难以在云端长期稳定运行

Hermes 的答案是：不只是更好地"处理一次任务"，而是构建一个能持续运行、持续学习、持续进化的 Agent 系统。

Slogan "The agent that grows with you"就是这个哲学的压缩表达。

2.3 发布与爆发：两个月，从零到近十万星

2026 年 2 月 25 日，Hermes Agent v0.2.0 正式发布。

这个时间点选得很准。恰好是 OpenClaw 把 Agent 概念在中文圈带火之后不久，很多人已经知道"Agent 是什么"，正好在找下一步。

发布后的增长速度是极端的。根据不同时间点的报道数据拼接：

  • 发布初期：800 星
  • 一个月内：接近 5 万星
  • 两个月内：约 7-9 万星，部分报道显示峰值接近 10 万
  • 贡献者：约 180-317 人（不同时间节点的快照，整体趋势上升）
  • Fork：6500+
  • 提交数：约 3000-3500 次

版本迭代速度同样惊人：从 v0.2.0 开始，几乎每周一个大版本，快速迭代到 v0.8.0 → v0.9.0，平均不到一周一个显著功能更新。

这个节奏反映的不只是技术积累，也是社区形成正反馈时的飞轮效应——星数越高，关注的开发者越多，PR 越多，迭代越快，又带来更多星数。

2.4 核心技术架构：用人话解释 Hermes 到底是怎么工作的

Hermes 的架构设计有几个核心概念，我尽量用不需要编程背景也能理解的语言解释：

闭环学习机制：它是怎么"越用越聪明"的？

这是 Hermes 最核心也最差异化的设计。

想象这样一个过程：
1. 你让 Hermes 帮你做一件事
2. 它做完了
3. 它会自动判断：这件事的做法，值得记下来吗？
4. 判断标准包括：调用工具超过 5 次、中途出了错自己修复了、你手动纠正过它、找到了比上次更快的方法……
5. 如果满足条件，它就自动生成一段"技能"（Skill）——可以理解成一份操作手册
6. 下次遇到类似的任务，它直接调用这个技能，不用重头摸索

更重要的是，这个技能不是一次性写死的，它会随着后续更好的实践"打补丁"更新，而不是每次都推倒重来。

这就是"闭环学习"。它不是真的像人类大脑那样学习，而是通过经验沉淀 → 技能生成 → 技能复用 → 技能优化这条链路，让 Agent 看起来越用越熟练。

四层记忆架构：为什么它记得住？

Hermes 把记忆系统拆成四层，每层管不同的事：

层级 存什么 特点
第一层（核心记忆） 每次都要用的关键信息（比如你的名字、常用偏好） 容量严格限制，逼你筛选什么最重要
第二层（对话存档） 所有历史对话，支持全文搜索 不直接载入上下文，按需检索
第三层（技能库） 自动生成的操作流程 按需调用，不占用日常资源
第四层（长期偏好） 用户的领域知识、长期习惯 适合长期当个人助理

这种分层设计解决了一个核心矛盾：记太多会让上下文拥挤、模型变迟钝；但什么都不记又没法"进步"。四层记忆的逻辑是：把不同粒度的记忆分开存放，用的时候按需取用，而不是把所有东西都塞进一次对话的上下文窗口里。

子 Agent 委托：让复杂任务分工完成

遇到大任务时，Hermes 可以把工作"委托"给多个并行运行的子 Agent。

  • 每个子 Agent 有自己的独立工作空间，不共享父 Agent 的上下文（防止信息污染）
  • 最多可以 3 个子 Agent 同时运行
  • 委托深度最大两层（父 → 子 → 孙，不能再深了，防止失控）
  • 子 Agent 完成后，结果汇报给父 Agent

上下文压缩：防止它"越聊越忘"

对话一长，Token 就会超限，模型开始忘掉前面的内容。Hermes 的应对方案是四阶段压缩：

  1. 先删掉旧的工具调用结果（最占空间、最不重要的内容）
  2. 确定哪些内容必须保留（最近的对话 + 开头的核心信息）
  3. 用一个更便宜的小模型对中间的对话做结构化摘要（目标、进度、关键决策、下一步）
  4. 修复摘要后可能破碎的工具调用配对

关键是：下次又触发压缩时，不是重新总结，而是在上一次摘要基础上增量更新。这保证了跨多次压缩的信息连续性。

定时调度：它可以在你睡觉的时候工作

Hermes 内置了 Cron 系统。你可以用自然语言设置任务：比如"每天早上 8 点总结昨天的新闻并发到 Telegram"。它会自动在指定时间运行，完成后推送结果。如果没有新内容要报告，它会发一个"静默标记"，不打扰你。

多平台接入

不绑定在你的电脑上。它可以跑在云端服务器（5 美元的 VPS 就够），然后通过 Telegram、Discord、Slack、WhatsApp、Signal 等 14+ 个平台和你对话。每个平台的对话记忆是共享的——你在 Telegram 说过的事，它通过 Slack 也记得。

2.5 关键事件：几个不能绕过的时间节点

2026年4月10日——小米 MiMo 官方接入

Xiaomi 的大模型 MiMo 宣布正式支持 Hermes Agent 框架，并推出限免活动。这被很多报道解读为"第一个国产大模型官方集成 Hermes 的案例"，象征意义大于实用意义，但这表明 Hermes 开始进入国内企业级生态。

与此类似，腾讯云和阿里云也陆续宣布对 Hermes Agent 的官方支持。

从 OpenClaw 迁移

Hermes 官方 README 里有一条值得关注的功能：hermes claw migrate——一键把 OpenClaw 的设置、记忆文件、技能和 API Key 全部迁移到 Hermes。

这一步的信号很明确：Hermes 没有假装 OpenClaw 不存在，而是主动承接它的用户。它不是在竞争，而是在说"你已经在用 OpenClaw 建立的工作习惯，可以无缝带过来"。这是一种非常有效的用户获取策略。

抄袭争议

这是必须提到的负面事件。

一家名为 EvoMap 的公司指控 Hermes Agent 的"闭环学习"核心机制与它们的 Evolver 引擎高度相似，甚至声称是直接抄袭。这个争议目前尚未有明确法律结论，但它直接指向了 Hermes 最核心的差异化能力——"自进化"到底是真正的原创设计，还是在已有工程成果上重新包装？

这个争议不能简单忽略，因为如果属实，它会动摇整个 Hermes 价值主张的根基。目前的处理方式是：在报告中客观呈现，不做定性判断。

三、横向分析：Hermes 在整个 Agent 赛道里，到底处于什么位置？

3.1 最重要的对比：Hermes vs OpenClaw

这两个项目被放在一起比的频率最高，但它们其实不是同一条赛道上的选手。

维度 OpenClaw Hermes Agent
核心定位 多平台接入的 Agent 工作入口 自进化的持续工作者
记忆能力 MEMORY.md + 会话记忆 四层记忆架构 + 全文搜索
学习能力 无自动技能生成 闭环学习 + Skill 自动沉淀
生态规模 13,000+ 技能（绝对领先） 70+ 技能（差距巨大）
部署方式 本地为主 + 多平台接入 云端 VPS + 多平台
技术门槛 相对容易上手 相近，但初期配置更多
GitHub 星数 35+ 万（成熟项目） 8-10 万（快速增长）
商业路径 开源社区驱动 开源 + 企业级商业化

结论：它们是互补关系，不是非此即彼。

  • OpenClaw 更像企业/个人的 Agent 工作基础设施——生态庞大，接入成本低，即装即用
  • Hermes 更像会自我进化的长期工作伙伴——越用越强，适合有长期重复工作流的场景

Hermes 在官方 README 里写"从 OpenClaw 迁移"，不是宣战，而是说："你在 OpenClaw 养成的习惯，我全盘接收。"

3.2 AutoGPT：第一代 Agent 的警示

AutoGPT 是 2023 年 AI Agent 赛道的第一次爆发，彼时星数飞速增长，但随后用户发现：任务规划能力很强，但执行往往失控，长任务很难稳定完成，而且每次都要从头开始。

它的失败不是技术失败，而是定位失败：把"自主规划"当成了 Agent 的全部，忽略了"可靠执行"和"持续工作"同样重要。

Hermes 吸取了这个教训，把重点放在持续运行的稳定性和经验积累上，而不是一次性的任务规划炫技。

3.3 LangChain / LangGraph：框架型 vs 产品型

LangGraph 和 Hermes 代表了两种完全不同的思路：

  • LangGraph：给工程师提供积木，让你自己搭 Agent 系统。高度可控，但需要编程能力，适合企业定制化开发
  • Hermes：直接是一个可运行的 Agent 产品，你只需要配置和使用，不需要从零搭系统

用一个比喻：LangGraph 是乐高，Hermes 是已经组装好的机器人。

3.4 Claude Code：垂直 vs 通用

Claude Code 是 Anthropic 专门为编程场景打造的 Agent，深度集成在代码工作流里——它读你的代码库、写代码、跑测试、提 PR。

Hermes 是通用型 Agent，不专注于任何特定场景。

两者有功能重叠（都能做代码相关任务），但定位不同：Claude Code 追求"编程场景的最强体验"，Hermes 追求"通用场景下越用越强的持续助手"。

3.5 n8n / Dify / Coze：工作流平台 vs 自主 Agent

这类产品更像"自动化工作流"——你画出流程图，系统按图执行。

Hermes 更像自主决策的 Agent——你告诉它目标，它自己决定怎么做，并且从每次做法中积累经验。

前者是流程的执行者，后者是任务的主动承担者。技术门槛和适用场景都不同。

四、横纵交汇洞察：Hermes Agent 真正意味着什么？

4.1 它的出现是一个信号

Hermes Agent 的爆火，不只是"又一个好工具火了"，更像是在宣告一件事：

AI Agent 赛道的竞争逻辑正在从"功能更多"转向"越用越强"。

第一代 Agent 竞争的是：我能接哪些平台，能做哪些任务。
Hermes 代表的下一代竞争是：做完任务之后，我变得更好了吗？

这个转变的底层逻辑是：
- 当所有 Agent 都接了同样的 API、连了同样的平台之后，真正拉开差距的是经验积累能力
- 用户真正想要的不是一个聪明的工具，而是一个越来越懂自己、越来越适配自己的工作系统

从这个角度看，"自进化"不只是 Hermes 的产品特性，而是它对整个赛道提出的一个问题：AI Agent 应该是静态工具，还是动态成长的系统？

4.2 为什么它和 OpenClaw 放在一起最能说明问题

OpenClaw 和 Hermes 合在一起，可以描绘 AI Agent 赛道的两个阶段：

OpenClaw 代表的是第一个问题被回答了：
"AI 可以持续工作、接入多平台、完成各种任务吗？" ——可以。

Hermes 在问下一个问题：
"AI 可以在工作中积累经验、沉淀能力、越用越好吗？" ——它在尝试。

这也是为什么 Hermes 官方第一步就做"从 OpenClaw 迁移"的功能。它承认 OpenClaw 已经验证了市场，自己是站在这个基础上往前走一步。

4.3 它的优势和风险都很明确

优势：
1. 理念差异化：闭环学习是目前开源 Agent 里最清晰的"自进化"实现
2. 云端原生：VPS 上能跑，适合 24×7 持续工作
3. 迭代速度极快：2 个月不到完成了 7-8 个主要版本
4. 多平台无缝：跨平台共享记忆是真实的工程能力
5. Nous Research 模型生态背书：有真实的开发者社区基础

风险：
1. 生态差距悬殊：70+ vs OpenClaw 的 13,000+ 技能，短期内很难弥合
2. Nous Research 的 Web3 背景：Paradigm 领投引发一部分用户的信任顾虑，担心项目走向代币化商业化
3. 抄袭争议：EvoMap 对"闭环学习"核心机制的指控，若属实，将动摇其核心差异化叙事
4. "越用越强"的实际效果：现阶段技能生成质量参差不齐，自进化的实际效果和宣传叙事之间仍有距离
5. 长时运行的稳定性问题：状态漂移、上下文管理失控等工程问题在实际使用中仍存在

4.4 会不会成为"下一个 AutoGPT"（热了又冷）？

这是一个合理的担忧。AutoGPT 当年也是几十万 GitHub 星，后来热度大幅消退。

Hermes 和 AutoGPT 有一个关键区别：它解决的问题更具体，用户价值更可测量。

AutoGPT 失败的核心原因是"炫技有余，实用不足"——它让人看到了 AI 自主规划的可能性，但执行结果不可靠到让人难以真实依赖。

Hermes 的"闭环学习"如果真的有效，用户会在使用中感受到真实的效果改善——这是一个可以被验证、被口碑传播的价值。这使它的长期生命力比 AutoGPT 更有可能持续。

但最终能否跑赢，还是要看：
- 技能生态能否快速补上
- 抄袭争议如何解决
- 随着 OpenClaw 本身也在迭代，差异化是否能持续

4.5 长期 Hermes vs OpenClaw，谁更可能成为主流？

不是二选一的问题，更可能是生态分工的问题。

OpenClaw 的定位更像"Agent 时代的操作系统"，它提供的是更宽泛的基础设施和生态。Hermes 的定位更像"会学习的专属助手"，它提供的是更深度的个性化适配。

两者都有可能同时成立：
- 企业部署、多系统接入、标准工作流 → OpenClaw
- 个人助手、长期知识沉淀、持续进化场景 → Hermes

当然，这个判断的前提是 Hermes 的"自进化"能在实际使用中真正兑现它的承诺。

五、结论：Hermes Agent 是什么，又不是什么

它是什么：

一个认真尝试回答"AI Agent 能不能越用越强"的开源项目。它在技术架构上有真实的差异化设计，在增长速度上展现了市场对这个方向的需求，在迁移兼容性上展现了产品成熟度。

它不是什么：

它不是 OpenClaw 的替代品，也不是已经证明"自进化"行得通的终态产品。它是这个方向上一个非常有价值的早期探索——速度很快，理念很清晰，但生态差距和若干争议说明它还在成长中。

一句话总结这件事：

OpenClaw 告诉大家"AI Agent 可以持续工作"；Hermes Agent 在问"AI Agent 能不能越工作越厉害"——这个问题值得被认真对待。
